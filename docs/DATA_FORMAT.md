# Data Format Specification

## Input Format

The application reads **CSV files** containing biomechanical time-series data from hip exoskeleton experiments.

## Column Specification

### Required Columns

These columns must be present (or mapped via Column Mapping) for the app to function:

| Column Name | Type | Unit | Description |
|-------------|------|------|-------------|
| `Time_ms` | numeric / datetime | ms or s | Timestamp for each sample |
| `imu_LTx` | float | degrees | Left hip joint angle |
| `imu_RTx` | float | degrees | Right hip joint angle |
| `imu_Lvel` | float | deg/s | Left hip angular velocity |
| `imu_Rvel` | float | deg/s | Right hip angular velocity |

### Optional Columns

These columns enable additional features when present:

| Column Name | Type | Unit | Used By |
|-------------|------|------|---------|
| `M1_torque_command` | float | Nm | Right motor torque command | Analyzer, Report |
| `M2_torque_command` | float | Nm | Left motor torque command | Analyzer, Report |
| `raw_LExoTorque` | float | Nm | Raw left actuator torque | Analyzer, Filter-Delay |
| `raw_RExoTorque` | float | Nm | Raw right actuator torque | Analyzer, Filter-Delay |
| `filtered_LExoTorque` | float | Nm | Pre-filtered left torque | Analyzer |
| `filtered_RExoTorque` | float | Nm | Pre-filtered right torque | Analyzer |
| `L_P`, `L_D` | float | Nm | Left PD controller terms | Analyzer |
| `R_P`, `R_D` | float | Nm | Right PD controller terms | Analyzer |
| `HumanL_Hip_tau` | float | Nm | Human left hip torque | Utils (power calc) |
| `HumanR_Hip_tau` | float | Nm | Human right hip torque | Utils (power calc) |
| `tag` | string | -- | Motion type label (e.g., "walk", "run", "sit") | All tabs |

## Time Axis Handling

The `make_time_axis()` function auto-detects the time format:

1. **Numeric, interval 1-1000** -> treated as milliseconds, converted to seconds
2. **Numeric, interval < 1** -> treated as seconds directly
3. **Datetime string** -> parsed and converted to elapsed seconds
4. **Fallback** -> row index used as time (seconds)

All time values are normalized to start from 0.

## Column Mapping

Different labs/devices may use different column names for the same signals. The app handles this through interactive column mapping.

### How It Works
1. On first load of a new data format, the app prompts with a **Column Mapping Dialog**
2. Match your dataset's column names to the expected names via dropdowns
3. Mapping is saved to `.column_mapping.json` in the project root
4. Subsequent loads with the same column names use the saved mapping automatically

### Auto-Detection Synonyms
The app recognizes common naming variants automatically:

| Expected | Also Recognized |
|----------|----------------|
| `Time_ms` | `time`, `time_s`, `timestamp`, `Time` |
| `imu_LTx` | `left_angle`, `L_angle`, `hip_L` |
| `imu_Lvel` | `left_vel`, `L_vel`, `hip_L_vel` |

### Adding Support for New Data Sources
1. Place CSV files in your data folder
2. Load the file -- Column Mapping dialog will appear if names don't match
3. Map each required column to the corresponding column in your data
4. The mapping is persisted automatically

If your data requires pre-processing (unit conversion, resampling, etc.), add a conversion function in `src/utils.py`.

## Output Formats

| Output | Format | Location | Generated By |
|--------|--------|----------|--------------|
| Figures | PDF (300 DPI) | `data_output/output/` | Analyzer, Gait Cycle, Filter-Delay |
| Summary Report | CSV | `data_output/output/<name>.csv` | Report Tab |
| Averaged Report | CSV | `data_output/output/<name>_avg.csv` | Report Tab |
